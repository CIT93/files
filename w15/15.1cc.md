The goal of this challenge is to move from a generic error object display to a specific, human-readable message.

This happens because the **`error.message`** property in your initial handling code is not correctly accessing a user-friendly string; it's likely trying to stringify the entire Geolocation error object, which is not what we want to show the user.

### The Challenge: Provide a Meaningful Message

The object passed to the Geolocation error callback (`error`) has a property called **`code`** which is a number representing the failure type (e.g., permission denied, timeout). We must use this code to create a clear message *before* wrapping it in a final `Error` object for rejection.

**Your Task:**

2.  Your solution must check the numeric value of **`error.code`** to generate a meaningful `errorMessage` string for the three scenarios:
      * **Permission Denied:** The user must be told to enable location services.
